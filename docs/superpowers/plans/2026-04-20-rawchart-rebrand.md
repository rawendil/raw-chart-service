# RawChart Rebrand Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rename the project from "Chart Service" / `chart-service` to "RawChart" / `rawchart-service` across user-facing text (README, Swagger title, HTTP root response, log metadata) and technical identifiers (npm package, Docker image, Kubernetes resources, container names, Postgres database name). API paths and repo directory are unchanged.

**Architecture:** This is a mechanical rebrand. No runtime behavior changes. Each file is an independent edit; tasks are grouped by file. Verification is by `grep` residual-references check and `npm run build`. No new source files; no new tests.

**Tech Stack:** TypeScript, Express, Swagger (`swagger-jsdoc`, `swagger-ui-express`), Winston logger, PostgreSQL, Redis, Docker Compose, Kubernetes manifests, npm.

**Spec:** [docs/superpowers/specs/2026-04-20-rawchart-rebrand-design.md](../specs/2026-04-20-rawchart-rebrand-design.md)

**Naming reference:**
- Brand (user-facing): **`RawChart`**
- Technical slug (kebab-case): **`rawchart-service`**
- DB / snake_case slug: **`rawchart_service`**
